# Changelog

## [0.5.5] - 2026-07-23

### Fixed

- Примусове перемикання WAN тепер підтверджує фактичний RouterOS state перед повідомленням про успіх. Кнопка наступного WAN визначається за активним main-маршрутом, тому після переходу на BITE показує LMT.

## [0.5.4] - 2026-07-23

### Added

- Sticky dual-WAN failover: primary WAN перемикається на резервний після трьох невдалих циклів без автоматичного повернення, з ручним переходом на конкретно вказаний наступний WAN.

Додано timeline-графік періодів, коли primary був LMT або BITE, та автоматичні повторні підключення до RouterOS кожні 5 секунд, коли API недоступний.

## [0.5.3] - 2026-07-22

### Fixed

- Технічний реліз для перевірки живого потоку автооновлення (0.5.2 → 0.5.3) після фіксу Quasar Dialog, функціональних змін немає

## [0.5.2] - 2026-07-21

### Fixed

- Зареєстровано Quasar Dialog у main.js — автооновлювач мовчки падав при спробі показати діалог "Доступне оновлення" (useUpdater() викликає $q.dialog, а був зареєстрований лише Notify), тож оновлення до 0.5.1 ніколи не пропонувалось користувачу

## [0.5.1] - 2026-07-21

### Changed

- Оновлено @7n/tauri-components до 0.13.9 — AgentDialog: лейбл моделі, видимість tier-селектора, продовження розмови при зміні агента, усунення гонки acp_spawn_agent/acp_prompt, коректна самоідентифікація моделі; fix(ci): changelog-release n-cursor замість стороннього @nitra/cursor

## [0.5.0] - 2026-07-20

### Added

- Кнопка "Відновити авто-перемикання" в діагностичному режимі — реверсує "Зафіксувати BITE": повертає LMT як primary (main/to_WAN1), BITE лишається primary лише для to_WAN2 як і до фіксації, і знову вмикає DUALWAN-health scheduler. Раніше відновлення після ручної фіксації BITE вимагало ручних команд у RouterOS-терміналі.

## [0.4.2] - 2026-07-18

### Added

- Увімкнено devtools у зібраному застосунку (Cargo `tauri` feature) — тепер правий клік → Inspect Element відкриває справжній Web Inspector (консоль, мережа) навіть у релізній збірці, щоб можна було діагностувати такі речі як тиху помилку перевірки оновлення.

## [0.4.1] - 2026-07-17

### Fixed

- Технічний реліз для перевірки живого потоку автооновлення (0.4.0 → 0.4.1), функціональних змін немає

## [0.4.0] - 2026-07-17

### Added

- Діагностичний режим: перевірка доступності RouterOS API, стану scheduler-а та поточного primary WAN кожні 5с, історія подій, звіт для чату та виправлення підрахунку ping-timeout у DUALWAN-health

## [0.3.1] - 2026-07-16

### Fixed

- Виправлено занадто великий заголовок у зібраному застосунку (0.3.0) — специфічність CSS тепер не залежить від порядку бандлингу. У тайтлі вікна тепер видно версію застосунку.

## [0.3.0] - 2026-07-16

### Added

- Застосунок сам перевіряє наявність нової версії (при старті й раз на годину) і пропонує встановити оновлення та перезапуститись
